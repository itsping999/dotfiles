# 云平台周报模板（V2）

## 模板标识

每份生成的周报必须包含以下标识（第一行标题后的 HTML 注释）：

```
<!-- TEMPLATE: WEEKLY_REPORT_V2 -->
```

## 标题格式

```
# 【{week_start} 至 {week_end}】云平台运行情况报告
```

示例：`# 【2026-06-06 至 2026-06-12】云平台运行情况报告`

## 版本说明

标题后紧跟引用块：

```
> _当前报告版本不为最终版，后续将不断迭代完善各类运行指标。_
```

## 章节结构与格式规范

以下章节按固定顺序排列，验证脚本按此顺序检查。每个章节的标题、表格列结构均为强制要求。

## 强制格式规则

1. **章节顺序固定**: 一 → 二 → 三 → 四 → 五，子章节顺序固定。
2. **标题层级固定**: `##` 用于主章节，`###` 用于子章节，`####` 用于三级章节。
3. **表格列数固定**: 每个表格的列数必须与模板定义一致。
4. **分隔线**: 标题与版本说明之间用 `---` 分隔，主章节之间用 `---` 分隔。
5. **无 5.9 章节**: 附录只到 5.8，不包含采集完整性章节。
6. **模板标识**: 文件必须包含 `<!-- TEMPLATE: WEEKLY_REPORT_V2 -->`。
7. **无源文件路径**: 正文中不得出现 `data/` 路径。
8. **建议观察**: 每个附录子章节必须包含 `#### 重点观察` 和 `#### 建议与改进措施`。

## 统计周期

- 固定为：`上周六 -> 本周五`（7 天，含首尾）。

## 输出路径

- 周报：`reports/weekly/<year>/<year>-W<week>.md`
- 索引：`reports/weekly/index.md`
