## 3DES-CBC Decryption

Uses `github.com/gogf/gf/v2/crypto/gdes` for Triple-DES CBC mode.

```go
import (
    "encoding/hex"
    "github.com/gogf/gf/v2/crypto/gdes"
)

const (
    DES_KEY     = "0123456789abcd0123456789"  // 24 bytes for 3DES
    DES_IV      = "12345679"                   // 8 bytes
    DES_PADDING = 1                            // PKCS5/PKCS7 padding
)

encryptedHex := "0cf23c53f0f22469ddb69c262b53c604..."
data, _ := hex.DecodeString(encryptedHex)

decrypted, err := gdes.DecryptCBCTriple(data, []byte(DES_KEY), []byte(DES_IV), DES_PADDING)
```

### Pitfalls
- Key must be exactly 24 bytes for Triple-DES (3 independent keys).
- IV must be exactly 8 bytes.
- `DES_PADDING = 1` is PKCS5/PKCS7. Value `0` = zero padding, `2` = no padding.

## RSA2048 Signing

Supports both PKCS#1 and PKCS#8 private key formats.

```go
import (
    "crypto"
    "crypto/rand"
    "crypto/rsa"
    "crypto/x509"
    "encoding/base64"
    "encoding/pem"
    "fmt"
)

func SignRSA(privateKeyPEM string, data []byte) (string, error) {
    block, _ := pem.Decode([]byte(privateKeyPEM))
    if block == nil {
        return "", fmt.Errorf("failed to decode PEM")
    }

    var pk *rsa.PrivateKey
    // Try PKCS1 first
    parsedKey, err := x509.ParsePKCS1PrivateKey(block.Bytes)
    if err == nil {
        pk = parsedKey
    } else {
        // Fall back to PKCS8
        pkcs8Key, err := x509.ParsePKCS8PrivateKey(block.Bytes)
        if err != nil {
            return "", fmt.Errorf("unsupported key format: %w", err)
        }
        var ok bool
        pk, ok = pkcs8Key.(*rsa.PrivateKey)
        if !ok {
            return "", fmt.Errorf("not an RSA private key")
        }
    }

    hashed := crypto.SHA256.New()
    hashed.Write(data)
    sig, err := rsa.SignPKCS1v15(rand.Reader, pk, crypto.SHA256, hashed.Sum(nil))
    if err != nil {
        return "", err
    }
    return base64.StdEncoding.EncodeToString(sig), nil
}
```

### Pitfalls
- Always try PKCS1 before PKCS8 — PKCS1 is more common in `.pem` files
  generated by OpenSSL without `-topk8`.
- `crypto.SHA256` vs `crypto.SHA1`: match the algorithm expected by the
  verifier. Alipay uses SHA256 (RSA2); ABC Bank uses SHA1.
